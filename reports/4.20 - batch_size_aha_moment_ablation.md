# Batch Size 与 Aha Moment 消融实验：Qwen2.5-3B PPO

> **实验背景**：TinyZero 复现 DeepSeek-R1 的 Aha Moment（模型在 RL 训练中自发学会 chain-of-thought 推理），此前在 batch=128 的实验中稳定观察到 Aha Moment，但在 batch=64 的实验中未出现或出现极晚。本实验通过控制 batch_size 和 mini_batch_size，验证 batch size 是否是 Aha Moment 出现的关键因素。

---

## 一、问题定义

### 什么是 Aha Moment

Aha Moment 指 RL 训练过程中，模型自发学会利用 `<think>` 标签进行推理的行为转变。表现为 response length 先下降（学会格式，回答变短）再上升（开始在 `<think>` 中进行计算推理，回答变长），伴随 test score 显著上升。

### 为什么关注 Batch Size

在前序实验中观察到：batch=128 的实验全部出现 Aha Moment，而 batch=64 的实验未出现或严重延迟。PPO 的策略梯度是对 batch 内样本的 advantage 加权平均，batch 越小，梯度估计噪声越大，模型越难从噪声中提取到稳定的学习信号。

### 本实验验证什么

在固定模型（Qwen2.5-3B）和其他超参的前提下，对比三组不同 batch_size / mini_batch_size 配置的训练动态，观察 Aha Moment 的出现时机、稳定性以及训练过程中梯度噪声（grad_norm、KL）的差异。

## 二、实验配置

### 对比组

| | 实验名 | batch_size | mini_batch_size | 更新次数/batch |
|--|--------|-----------|----------------|---------------|
| **A** | `ppo-3b` | 64 | 16 | 4 |
| **B** | `ppo-3b-bs64-mini32` | 64 | 32 | 2 |
| **C** | `ppo-3b-bs128-mini64` | 128 | 64 | 2 |

### 三组共同超参

| 参数 | 值 |
|------|----|
| 基座模型 | Qwen/Qwen2.5-3B |
| `adv_estimator` | GAE（`gamma=1, lam=1`） |
| `kl_coef` | 0.001 |
| `clip_ratio` | 0.2 |
| `entropy_coeff` | 0.001 |
| `ppo_epochs` | 1 |
| actor `lr` | 1e-6 |
| critic `lr` | 1e-5 |
| `max_response_length` | 512 |
| `rollout.n` | 1 |
| `total_epochs` | 15 |
| GPU | 2 × H100-96 |

### 各组差异（仅影响显存分配，不影响梯度计算）

| | A | B | C |
|--|---|---|---|
| `gradient_checkpointing` | False | True | True |
| `ref.param_offload` | False | False | True |
| `tensor_model_parallel_size` | 2 | 2 | 1 |
| `gpu_memory_utilization` | 0.4 | 0.4 | 0.25 |
| `val_batch_size` | 128 | 1312 | 1312 |
| `test_freq` | 50 | 25 | 25 |

> **注**：A 组 `val_batch_size=128`（远小于 B/C 的 1312），导致 A 组的 test score 评估样本较少，数值波动更大。A 组 `test_freq=50`，因此 test score 数据点比 B/C 更稀疏。

![actor/grad_norm](../assets/ppo-batchsize-minibatch-ahamoment-%20ablation/actor-grad-norm.png)

| | A（bs64-mini16） | B（bs64-mini32） | C（bs128-mini64） |
|--|-----------------|-----------------|-------------------|

- C 组 grad_norm 始终维持在 ~0.1-0.2，几乎无尖峰。
- A 组与 B 组的区别在于：A 组是**持续高噪声**，B 组是**偶发大尖峰但基线平稳**。

![actor/ppo_kl](../assets/ppo-batchsize-minibatch-ahamoment-%20ablation/actor-ppo-kl.png)

## 四、结论

**1. Batch size 决定 Aha Moment 能否出现。**

C 组（bs128）呈现经典的 Aha Moment 模式：response length 先降后升（step ~75 回升），test score 持续上升至 ~0.57。A 组（bs64-mini16）的 response length 在下降后未出现明确回升，test score 最终仅到达 ~0.44，未出现 Aha Moment。B 组（bs64-mini32）介于两者之间，response length 有回升但波动大。

**2. Batch size 越小，梯度噪声越大。**

A 组的 grad_norm 平均水平（~0.3-0.8）是 C 组（~0.1-0.2）的 3-4 倍，且尖峰频繁。A 组的 KL 在 step 200-300 出现密集尖峰（最高 ~0.018），对应同时期 test score 的停滞/回落（从 ~0.41 降至 ~0.34）。C 组的 grad_norm 和 KL 始终平稳。

**3. 在 batch size 相同（bs64）的前提下，mini_batch_size 影响训练稳定性。**

B 组（mini32，2 次更新）比 A 组（mini16，4 次更新）的 grad_norm 平均水平更低、Aha Moment 出现更早、最终 test score 更高（~0.55 vs ~0.44）。B 组虽有个别 grad_norm 尖峰（~1.4），但基线平稳；A 组的问题不在于尖峰，而在于**持续的高噪声**。

**4. Batch size 不仅影响收敛速度，也影响最终性能。**

A 组最终 test score（~0.44）明显低于 B/C 组（~0.55）。在 500 步训练范围内，bs64-mini16 的配置不足以让模型到达与 bs128 相同的性能水平。
