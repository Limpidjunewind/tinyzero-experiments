Background

DeepSeek-R1 / TinyZero: pure RL induces reasoning in small LLMs

The Claim

No CoT supervision needed

Rule-based reward alone causes models to spontaneously develop chain-of-thought inside <think> — no labelled reasoning data required.

Task & Reward

Countdown + 3-tier rule reward

Given 3–4 numbers, reach a target using +/−/×/÷. Reward: 0 (no tag) · 0.1 (valid format) · 1.0 (correct). <think> is never scored.

Aha Moment

Response length drops, then rises

The model finds a shortcut (short, high score), then spontaneously expands its reasoning. The length rebound is the observable signal of genuine strategy shift.

Motivation

Three questions we set out to answer

Q1 · Capacity

What model size is needed for reasoning emergence?

We test 0.5B, 1.5B, and 3B under identical hyperparameters at batch=64 — our GPU-constrained baseline (paper uses batch=256).

Q2 · Batch Size

What is the minimum batch size for Aha Moment?

We ablate batch size systematically to find the smallest configuration that triggers the response-length rebound.

Q3 · Stability

What causes entropy collapse — and can it be fixed?

At larger batch sizes, training can collapse catastrophically. We trace the root cause and identify a framework-level bug.

03  ·  The real culprit was hiding.
1
vLLM (rollout) outputs −∞ for padding log_prob; FSDP (training) outputs −87.9 Different kernels, different behavior after EOS — silently inconsistent.
2
PPO ratio = exp(−87.9 − (−∞)) = NaN → all weights corrupted One NaN token propagates through loss.mean() — irreversible. No hyperparameter can patch this.
04  ·  Fix the source, not the symptom.
FINDING · 04

Root cause is a kernel-level train-infer inconsistency (vLLM vs. FSDP log_prob on padding tokens) — real fix is Rollout Correction or log_prob clipping, not entropy_coeff tuning

Summary

Four takeaways

Capacity threshold

0.5B can't escape the reward plateau

Reasoning emergence requires at least 1.5B. 0.5B gets stuck with near-zero gradient signal from near-uniform reward distribution.

Batch size matters

batch=128 is the minimum for Aha Moment

Backed by ablation data. The paper's batch=256 choice is a necessary condition, not an arbitrary default.

Initialization shapes behavior

Cold start anchors think usage — result undetermined

SFT cold start keeps think non-empty, but training was cut at step 250. Whether it catches up to the base model with full training remains unanswered.

Root cause is framework-level

Collapse: train-infer inconsistency → NaN

vLLM and FSDP compute log_prob differently on padding tokens. entropy_coeff and KL tuning are patches — real fix aligns log_prob across both sides.
