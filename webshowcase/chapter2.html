Motivation · Why Study the Reward

Reward Is the Only
External Signal in RL

Every gradient in PPO comes from the reward. No labels, no demonstrations — what the model learns, how it learns, and how deeply it learns are all decided by how the reward function is written.

PPO training loop · only R(τ) connects to the outside world
1
external signal
No SFT data, no demonstrations — gradients come entirely from the reward
3B
small-model params
Limited capacity makes the model acutely sensitive to every reward design choice
1

Model, data, hyperparameters, hardware all fixed — only the reward moves

If the reward is the only bridge between "training signal" and "model behavior"—
then changing it should change everything the model learns. This is the hypothesis we set out to test.

Chapter I · Core Observation

Beneath the Numbers · Three Surprises

Underneath the tidy curves, three phenomena echo each other.

81.66%
"Wrong answers" actually computed the right value
The bottleneck isn't "can it compute" but "can it strictly use only the given numbers." Input was [4,8,2,39], yet the model quietly produced 3 and 10.
4.4 → 0.9%
Multiplication & division got crowded out
Proximity opens a "fast lane" for add/subtract. The +0.1 bonus v2 added for mult/div can't counter the 0.4 proximity pull.
−40%
v3 is 40% faster per step than v2
vLLM batch inference waits on the tail. v2's max median hits 1024; v3 is only 220 tokens — tail length, not mean, drives speed.

② and ③ share a root cause — both are side effects of the proximity score. ① looks like a separate thread, coming from a too-loose number constraint. The three threads will intersect unexpectedly in the next chapter.

Conclusion

Reward Balance Is a
Critical Design Parameter

This project investigated two open questions left by TinyZero: under what conditions does RL-induced reasoning emergence occur, and how much does reward function design affect what a model learns once it does? The two lines of investigation converge on a shared underlying principle.

When any reward dimension dominates the others, the model finds the cheapest path to maximise it — and strategy diversity collapses. Reward balance matters as much as reward strength. The reward design space in small-scale RLVR is far less explored than commonly assumed.
